Recently, I came across several questions one of our developers answered for a new user of EnterMedia digital asset management. I thought our answers might help other users learn a bit more about EnterMedia. Since we are an open source company, am sharing them here. I'll have to post questions and answers like this more often:

What is the number of simultaneous concurrent sessions that the EnterMedia System can handle?

With 100 servers each with 1000 simultaneous concurrent sessions would get you 100k sessions. These servers can be load balanced in the same way Google load balances search requests across multiple servers. When there are edits or uploads those changes are propagated across the servers so if one server goes down the others can fill in.

Can the EnterMedia System handle high volume streaming?

Yes. We recommend using the Amazon delivery network along with
http://www.adobe.com/products/httpdynamicstreaming/

Can the EnterMedia System auto adjust data bandwidth in media streams to compensate for any bottlenecking?

Yes using the Adobe flash player

Is there a standard "build playlist" feature with easy drag-n-drop interface?

We support drag and drop in the browser using jQuery.

Does EnterMedia use a database? If so or if not, what is the logic and advantages to this type of setup?

EnterMedia has a pluggable data storage system. For distributed load balanced systems, we use Apache Lucene to search and update live indexes. We store all videos, images and data on the file system for backup, scalability and deployment reasons. Files are organized into catalogs of assets stored within a top level folder. Multiple catalogs to be served from the same server and mounts are used to spread a catalog across disk volumes.
