There is a little setup work needed to enable per  user themes. In your _site.xconf you would need to change the  "themeprefix" variable to something like this:

<property name="themeprefix">/${applicationid}/themes/$[userthemepack]</property>
 
Note the {} are used to resolve to other properties and the [ ]  are used to pull user profile settings. 
 
You  can setup variables on a per user or per settings group. A settings  group can used to assign themes on a per catalog basis. Each use would  then be assigned a settings group for that catalog. 
 
Creating a Theme:
 
We recommend starting with a copy of  one of the applications to make a custom theme. /eml is a good starting  point. We have recently made the move to jQuery UI for most CSS themes.  /eml is a new theme that uses jQuery and we expect to finalize by July.
 
1. Create a folder for your theme:
 
/theme/styles/content.css <-- Standard Font sizes, colors
/theme/styles/layout.css  <-- spacing and layout for navigation, todo: clean out and combine  useful parts from yaml basemod_grids and yamlbase.css
 /theme/styles/theme.css <-- This has custom overrides/tweaks for font colors etc. It is loaded last
 
2. Use the base/emfrontend/theme/jqueryui to store all custom UI themes
 
3. Use _site.xconf to include the extra styles sheets you need. Dont use @import url(xxx.css);
 
4. ui-components.js has common widgets: 
       uibutton uidropdown uicalendar uipanel uimessagebox uitabs
